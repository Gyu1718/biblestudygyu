# Input Workflow

이 문서는 새 책을 아카이브에 추가할 때 따르는 작업 순서입니다.

## 1. 기본 원칙

이 프로젝트는 원문 저장소가 아니라 연구 색인입니다. 책 파일을 직접 올리지 않고, 책에서 추출한 다음 정보를 직접 작성해 공개합니다.

- 서지정보
- 목차
- 장별 요약
- 핵심 주장
- 주제 태그
- 학자 태그
- 성경 본문 태그
- 짧은 인용 위치

## 2. 책 한 권 추가 순서

### 1단계: 원문 파일은 로컬에 보관

PDF, EPUB, OCR TXT 파일은 공개 저장소에 올리지 않습니다. 원문 파일은 개인 보관 위치에서만 관리합니다.

### 2단계: 책 ID 정하기

책 ID는 영문 소문자와 하이픈만 사용합니다.

예시:

- `calvin-institutes`
- `berkhof-systematic-theology`
- `bavinck-reformed-dogmatics`
- `watson-shorter-catechism`

한 번 정한 ID는 가급적 바꾸지 않습니다.

### 3단계: 템플릿 복사

`templates/book-entry.template.json`을 복사해 책 정보를 작성합니다.

필수 입력 항목:

- `id`
- `title`
- `author`
- `originalAuthor`
- `tradition`
- `category`
- `language`
- `summary`
- `topics`
- `chapters`

### 4단계: 장별 요약 작성

각 장마다 다음 정보를 입력합니다.

- 장 제목
- 1-3문장 요약
- 핵심 주장
- 관련 주제
- 관련 학자
- 관련 성경 본문
- 인용 위치

원문을 길게 복사하지 말고, 반드시 직접 요약합니다.

### 5단계: 주제 태그 연결

`data/taxonomy.json`을 먼저 확인하고, 가능한 한 표준 주제 태그를 사용합니다.

새 주제가 필요하면 `data/taxonomy.json`에도 추가합니다.

### 6단계: 학자 데이터 연결

새 학자가 나오면 `data/authors.json`에 추가합니다. 이미 있는 학자는 같은 이름과 ID를 사용합니다.

### 7단계: 성경 본문 연결

중요 본문이 나오면 `data/passages.json`에 추가합니다.

## 3. 한 권 처리 체크리스트

- 원문 파일을 공개 저장소에 올리지 않았다.
- 책 ID를 정했다.
- 서지정보를 입력했다.
- 책 전체 요약을 직접 작성했다.
- 목차를 장 단위로 나누었다.
- 장별 요약을 직접 작성했다.
- 핵심 주장 2-5개를 정리했다.
- 표준 주제 태그를 붙였다.
- 관련 학자를 연결했다.
- 관련 성경 본문을 연결했다.
- 인용 위치만 짧게 기록했다.
- 긴 원문이 들어가지 않았는지 확인했다.
- JSON 문법 오류가 없는지 확인했다.

## 4. 우선 입력할 책 추천 순서

1. 칼빈, 『기독교 강요』
2. 벌코프, 『조직신학』
3. 바빙크, 『개혁교의학』
4. 토머스 왓슨, 『웨스트민스터 소요리문답 해설』
5. 하이델베르크 요리문답 또는 웨스트민스터 신앙고백 관련 자료

## 5. 장별 요약 작성 기준

좋은 장별 요약은 다음 세 가지를 포함합니다.

1. 이 장이 다루는 질문
2. 저자의 핵심 주장
3. 다른 주제와 연결되는 지점

## 6. 운영 리듬

한 번에 책 전체를 끝내려 하지 말고, 다음 단위로 나누어 업데이트합니다.

- 1회 업데이트: 책 1권의 기본 정보
- 2회 업데이트: 목차 전체
- 3회 업데이트: 장별 요약 3-5개
- 4회 업데이트: 주제 태그 보강
- 5회 업데이트: 성경 본문 색인 보강
- 6회 업데이트: 역사·주제·본문 연결 보강
