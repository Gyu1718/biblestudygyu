# Passage Depth Schema

`data/passages.json`의 본문 카드는 단순 본문 색인이 아니라 설교, 교리교육, 연구에 바로 활용할 수 있는 본문 연구 노트로 확장합니다.

## 1. 기본 구조

```json
{
  "id": "romans-9",
  "reference": "로마서 9장",
  "book": "로마서",
  "testament": "New Testament",
  "summary": "본문의 신학적 연결 요약",
  "topics": ["예정론", "선택"],
  "relatedBooks": ["calvin-institutes"],
  "keyQuestions": [
    "본문이 던지는 핵심 신학 질문"
  ],
  "traditionUses": [
    "개혁파 정통: 이 본문을 어떻게 사용하는가",
    "신정통주의 비교: 이 본문을 어떤 관점에서 다시 묻는가"
  ],
  "sermonUses": [
    "설교에서 사용할 수 있는 방향"
  ],
  "researchUses": [
    "논문, 글쓰기, 교리교육에서 사용할 수 있는 방향"
  ]
}
```

## 2. 필드 설명

| 필드 | 설명 |
|---|---|
| `keyQuestions` | 본문이 던지는 핵심 신학 질문입니다. 본문을 읽을 때 무엇을 물어야 하는지 정리합니다. |
| `traditionUses` | 개혁파 정통, 바빙크, 벌코프, 바르트 등 전통 또는 학자별 활용 방식을 정리합니다. |
| `sermonUses` | 설교와 교회교육에서 본문을 어떻게 사용할 수 있는지 정리합니다. |
| `researchUses` | 논문, 브런치 글쓰기, 교리교육, 비교표 작성에서 본문을 어떻게 활용할지 정리합니다. |

## 3. 작성 원칙

- 본문 자체를 길게 인용하지 않습니다.
- 본문을 교리의 증거구절로만 축소하지 않습니다.
- 본문이 놓인 문맥, 교리적 사용, 설교적 활용을 구분합니다.
- 개혁파 정통과 신정통주의를 동일화하지 않습니다.
- 비교가 필요한 경우 바르트, 바빙크, 벌코프, 칼빈 등 구체적 기준점을 남깁니다.

## 4. 화면 표시

`js/passage-depth-enhance.js`가 본문 카드에 다음 접이식 섹션을 자동으로 붙입니다.

```txt
본문 연구 노트
- 핵심 질문
- 전통별 활용
- 설교 활용
- 연구 활용
```

이 스크립트는 기존 `app.js`를 직접 수정하지 않고, 본문 카드가 렌더링된 뒤 보조 섹션을 덧붙입니다.
